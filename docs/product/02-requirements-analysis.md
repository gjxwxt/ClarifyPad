# 02. 深入需求分析

## 1. 目标用户

### 1.1 首批用户

推荐优先服务以下人群：

1. **产品经理 / 项目经理**
   - 经常把模糊想法变成需求、任务、会议结论。
   - 痛点是表达不够清楚，容易被研发/设计/运营误解。

2. **运营 / 市场 / 销售**
   - 经常写私域消息、客户回复、活动说明、跟进记录。
   - 痛点是信息多、场景碎、需要语气合适。

3. **管理者 / 创业者**
   - 经常布置任务、反馈问题、做决策同步。
   - 痛点是临时想法多，但发出去前需要清晰、克制、可执行。

4. **客服 / 客成 / 售后**
   - 高频回复、需要统一口径。
   - 痛点是重复性强，但又不能机械。

5. **AI 高频用户**
   - 经常给 ChatGPT / Claude / Cursor / Manus 等工具写 prompt。
   - 痛点是 prompt 写得不够清楚导致输出不稳定。

### 1.2 不建议作为首批用户

1. 纯长文作者：他们更需要编辑器、知识管理和长文结构。
2. 只想纠错的人：Grammarly / 系统工具已足够。
3. 打字极快且表达能力强的人：除非有强记忆和自动化价值。
4. 对隐私极敏感且无法使用任何云端 AI 的企业用户：可以放到后续企业版。

## 2. 核心场景

### 场景 A：工作 IM 发送前优化

用户在飞书/微信/Slack 中想发一段话，但担心太啰嗦、太冲、太散。

流程：
1. 快捷键唤起输入中间层；
2. 输入粗糙想法；
3. AI 输出“直接发送版 / 委婉版 / 更短版”；
4. 用户点击确认；
5. 文本插入当前聊天框。

### 场景 B：把想法变成任务描述

用户想给团队布置任务，但只想到一些散点。

AI 输出：
- 任务标题
- 背景
- 要做什么
- 不做什么
- 截止时间
- 验收标准
- 待确认问题

### 场景 C：把吐槽变成建设性反馈

用户原始输入可能带情绪：

> 这个页面真的太乱了，用户肯定找不到入口，而且昨天说的那个按钮又没改。

AI 输出：
- 客观问题描述
- 影响
- 建议
- 需要对方确认的点
- 降低冲突的表达版本

### 场景 D：把粗糙 prompt 变成高质量 prompt

用户输入：

> 帮我分析这个用户反馈，看看产品哪里有问题。

AI 输出：
- 角色
- 任务
- 输入材料要求
- 输出格式
- 分析维度
- 约束条件
- 示例

### 场景 E：确认内容沉淀

每次用户点击“确认插入”后，系统记录：
- 原始输入
- 最终插入文本
- 场景
- 使用的处理方式
- 用户是否二次编辑
- 时间、目标 App、可选标签

长期形成：
- 常用表达模板
- 用户语气偏好
- 常见任务类型
- 个人 prompt 库
- 工作复盘素材

## 3. 用户痛点

### 3.1 表达前的痛点

- 想法是碎的，不知道怎么组织。
- 知道自己想表达什么，但写出来不够清楚。
- 害怕表达太冲、太弱、太啰嗦。
- 临时打字容易遗漏背景、时间、验收标准。
- 不想切到 ChatGPT，因为打断当前工作流。

### 3.2 表达后的痛点

- 发出去才发现别人理解错了。
- 任务没有验收标准，后续返工。
- 同样的问题反复解释，不能沉淀。
- 自己说过什么、确认过什么，散落在各个软件里。
- AI 工具帮忙写了，但没有持续学习自己的风格。

## 4. 需求分层

### 4.1 基础需求：可用

- 快捷键唤起
- 光标附近弹窗
- 支持输入文字
- 支持调用 AI 生成优化结果
- 一键插入当前输入框
- 记录确认历史

### 4.2 进阶需求：好用

- 多种输出模式：精简、正式、友好、任务化、Prompt 化、邮件化
- 输出前展示信息缺口
- 允许用户继续补充
- 支持多版本对比
- 支持撤销插入
- 支持常用模板
- 支持个人词典/术语

### 4.3 高阶需求：离不开

- 根据目标 App 自动推断场景
- 根据聊天对象/项目自动调整语气
- 从历史确认内容中学习表达偏好
- 自动生成个人工作复盘
- 支持团队模板和组织口径
- 支持本地模型和企业私有化

## 5. 用户旅程

### 第一次使用

1. 安装应用；
2. 授权辅助功能/输入权限；
3. 设置快捷键；
4. 打开任意输入框；
5. 按快捷键；
6. 输入一句粗糙想法；
7. 看到 AI 输出；
8. 一键插入；
9. 获得“确实省事”的首次价值。

### 高频使用

1. 用户不再思考“怎么组织语言”，先把想法丢进弹窗；
2. AI 自动识别是回复、任务、需求、prompt、总结；
3. 用户只做确认和小修；
4. 系统记录确认内容；
5. 下次输出更接近用户风格。

## 6. 核心用户价值

### 6.1 对个人

- 更快组织表达；
- 更少沟通误解；
- 更稳的语气和专业度；
- 形成个人表达资产；
- 把每天的零散想法沉淀下来。

### 6.2 对团队

- 任务描述更清晰；
- 会议和 IM 信息更可追踪；
- 口径更统一；
- 新人更容易写出合格表达；
- 减少反复追问和返工。

## 7. 核心假设

### 假设 1：文字输入场景下也存在“粗糙输入 → 清晰输出”的强需求

验证方式：
- 找 20 个知识工作者访谈；
- 观察他们在 IM / 邮件 / 任务系统里是否经常反复修改；
- 让他们试用原型，记录是否愿意替代 ChatGPT 复制粘贴。

### 假设 2：信息缺口提醒比单纯润色更有价值

验证方式：
- A/B 测试：只给改写 vs 改写 + 缺口提醒；
- 统计用户补充信息次数、最终插入率、满意度。

### 假设 3：确认记录能形成长期粘性

验证方式：
- 看用户是否回看历史；
- 是否从历史中复用表达；
- 是否愿意让系统基于历史学习偏好；
- 7 日/30 日留存是否因记忆功能提升。

## 8. MVP 成功标准

建议 v0.1 的成功标准：

- 日均唤起次数 ≥ 8 次/活跃用户；
- AI 生成后插入率 ≥ 50%；
- 生成后用户二次编辑比例 ≤ 40%；
- 7 日留存 ≥ 25%；
- 用户主观评分 ≥ 4/5；
- 用户至少能说出一个“以前需要复制到 ChatGPT，现在不用了”的场景。
